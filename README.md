# Urban Lunch – Mobile Testing Project

This project focuses on manual mobile application testing practices applied to the Urban Lunch application.

The project demonstrates mobile UI validation, functional testing, checklist execution, user flow validation, and bug reporting using structured QA documentation techniques.

---

# Project Scope

The project was designed to validate the functionality, usability, and interface behavior of the Urban Lunch mobile application.

Testing activities included:

- Mobile Application Testing
- Functional Testing
- UI Validation
- User Flow Validation
- Checklist-based Testing
- Bug Reporting
- Mobile Interface Validation

---

# QA Activities Performed

- Executed manual testing scenarios on mobile application features
- Validated mobile interface behavior and usability
- Verified restaurant and order selection workflows
- Tested user interactions and navigation flows
- Executed checklist-based validation scenarios
- Identified and documented software defects
- Linked bug reports using JIRA
- Validated expected versus actual behavior
- Performed regression validation scenarios
